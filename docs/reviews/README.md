# Reviews & Validation

This directory stores outputs from the Validation (V) phase of the RQPIV workflow.

## Contents

| Artifact Type | Filename Pattern | Generated By |
|---------------|------------------|--------------|
| Code review | `code-review-{session}.md` | code-reviewer |
| Test report | `test-report-{session}.md` | test-automator |
| Security audit | `security-audit-{session}.md` | security-auditor |
| Documentation | `documentation-{session}.md` | documentation-writer |
| Final validation | `final-validation-{session}.md` | Orchestrator |

## Purpose

These artifacts ensure:

1. **Code Quality**: All changes meet quality standards
2. **Test Coverage**: Adequate testing exists
3. **Security Compliance**: No vulnerabilities introduced
4. **Documentation**: Changes are properly documented
5. **Audit Trail**: Record of all validation activities

## Report Types

### Code Review Report

Contains:
- Files reviewed
- Critical issues (must fix)
- Warnings (should fix)
- Suggestions (consider)
- Patterns compliance check
- Recommendation (APPROVE/CHANGES NEEDED)

### Test Report

Contains:
- Tests run and results
- Coverage metrics
- New tests added
- Failing tests with analysis
- Coverage gaps

### Security Audit

Contains:
- Vulnerability summary by severity
- OWASP compliance checklist
- Dependency vulnerabilities
- Authentication review
- Recommendations

### Final Validation

Combines all reports into single status:
- READY FOR MERGE
- READY WITH NOTES
- NEEDS REMEDIATION

## Quality Gates

Validation must pass:
- [ ] No critical code review issues
- [ ] All tests passing
- [ ] No critical/high security vulnerabilities
- [ ] Documentation updated

## Workflow

1. **Code Review**: Review all changes for quality
2. **Test Automation**: Run and generate tests
3. **Security Audit**: Check for vulnerabilities
4. **Documentation**: Update relevant docs
5. **Final Report**: Combine and determine status
